About

Ten-plus years operating production infrastructure in regulated, business-critical environments, including medical-device manufacturing: Linux, virtualisation, Kubernetes, observability, and backup/DR with verified restore — work that holds under change management and audit.

The through-line is constraining non-deterministic runtimes behind deterministic safety envelopes. Stochastic AI and agent loops are gated by behavioral contracts, tiered authority, and action budgets; failure paths fail closed; every action lands in an append-only, output-hashed evidence chain. Hazards are modeled with STPA before mechanism is chosen, and controls map directly to EU mandates (NIS2, CRA, DORA, the AI Act), backed by hands-on ISMS/BCM work and an ISO 27001 auditor qualification.

Bias: open source over closed, mostly Ubuntu and Red Hat; stack-agnostic where the AI integration runs over an API (local models plus the Claude, OpenAI, and Gemini APIs). The same discipline runs in a self-hosted homelab — governed execution, runtime control points, observability for LLM and agent workloads.

Regulatory Trust Boundaries

EU mandates treated as operational trust boundaries — each mapped to an enforced mechanism rather than a policy statement.

NIS2 / ISO 27001
Append-only, output-hashed audit ledgers as forensic evidence chains — SHA-256 per call, output body never persisted; qualified-signature (QES) verification for ISMS records. ISO 27001 auditor-qualified.
EU AI Act / CRA
Stochastic inference isolated to bound blast radius — Kubernetes Pod Security (restricted PSA, seccompProfile: RuntimeDefault, default-deny NetworkPolicy) for inference workloads, and bounded process execution (subprocess + rlimit) for agent skill contracts; capability/namespace isolation as an explicit extension point.
AI Act high-risk / Machinery
STPA hazard modeling — losses to hazards to unsafe control actions — and inference-boundary contracts settled before mechanism is chosen (6dof-ascent-sim, mission-assurance, platform-blueprint).
DORA
Operational resilience — verified-restore backup/DR, graceful-degradation and recovery-verification patterns, and fail-closed transports under degraded conditions.
GDPR
PII redaction at the telemetry boundary (OpenTelemetry collector) and secret redaction in audit records before anything is persisted.

Invariant Matrix

Core constraints of the two systems I maintain in the open, stated as invariants and verified against each repository.

agents Apache-2.0 · pre-1.0
Boundary
Authority boundary between models, agents, and tools; untrusted skill bundles load with contracts off, pinned by an immutable ref plus sha256.
Control
Behavioral contracts (pre/invariant/post) and action budgets (steps, tokens, wall-clock, tool-calls) on one guard gate: approve, reject, or require-approval, with pause/resume and a circuit breaker.
Isolation
Namespace-bound memory; role and attribute ACLs, MVCC version tokens, AES-256-GCM at rest; opt-in subprocess and rlimit skill execution.
Audit
OpenTelemetry events, Jensen-Shannon drift, and self-attesting run-provenance with an offline verification gate.
relay-shell Apache-2.0 · v0.1.0
Boundary
Scoped service account, no sandbox by design; HTTP bound to loopback behind a CIDR Caddy/TLS edge, with optional OAuth 2.1 (DCR, PKCE).
Control
Central runner, denylist-first policy, tiered authority 0 to 3 (open, guarded, readonly), privileged-cue flagging; the transport fails closed.
Isolation
Per-call timeout and output caps, bounded session count and buffers, idle and lifetime reaping; output bodies are never persisted.
Audit
Append-only JSONL, one line per call: SHA-256 of the output (never the body), byte length, exit code, tier, and request and client id; secrets redacted.

Observability & Verification

The telemetry and verification loop that keeps unattended execution legible and reversible.

Metrics & traces
OpenTelemetry into VictoriaMetrics, surfaced in Grafana; per-call timing and resource use captured. Agent action budgets bound wall-clock, tokens, steps, and tool-calls per run, separating deterministic gate overhead from stochastic inference time.
Drift
Jensen-Shannon distributional drift on agent behavior, emitted as structured OTel-ready events per step (agents).
Evidence
Append-only, output-hashed audit pipelines (relay-shell, nous MCP surface) and self-attesting run-provenance behind an offline verification gate (agents).
Regression
Property-based and fuzz suites gating lights-out execution loops — relay-shell fuzz tests, nous estimator property tests — run in CI; Wazuh SIEM across the homelab fleet.

Homelab

A self-run, all-open-source fleet of mostly Ubuntu hosts. Local LLM inference on both NVIDIA (llama.cpp and llama-swap) and AMD ROCm (Ollama), fronted by a self-built MCP gateway on Hetzner that drives PydanticAI agents, OSINT feeds, and CVE and EU-regulatory intel.

Underneath it: a single-node Talos Kubernetes cluster running VictoriaMetrics, OpenTelemetry, and Wazuh for observability and SIEM; ZFS with Sanoid snapshots; a Tailscale mesh. Same change-control and evidence discipline as the regulated environments.
